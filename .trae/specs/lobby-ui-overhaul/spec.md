# 大厅入口 UI 商业化重构 Spec

## Why
老板希望将原本简单的“输入大名和房号”的大门，升级为包含多款游戏集合、带有底部导航栏（大厅、开房、我的）的经典商业化房卡棋牌大厅界面（如截图所示）。虽然市面上有 Cocos 等重客户端开源项目，但修改和部署门槛极高。由于咱们项目采用的是现代化 React 技术栈，完全可以低成本、高效率地在网页端一比一复刻出这套商业大厂级的大厅 UI，实现免下载即玩。

## What Changes
- **废弃旧版单调的大门**：将前端登录页重构为拥有底部三个 Tab（大厅、开房、我的）的全屏沉浸式移动端布局。
- **大厅 Tab (Lobby)**：
  - 顶部栏：显示用户头像、昵称、房卡数量、商城图标。
  - 中心网格：展示 6 款经典棋牌游戏入口卡片（如“斗牛集合”、“三公集合”、“炸金花集合”等）。点击“斗牛集合”弹出真正的“输入房号/创建房间”对话框。
- **开房 Tab (Room/Guild)**：
  - 页面提示“加入公会后，等待会长开房”或提供快速建房按钮，配合刷新和复制链接功能。
- **我的 Tab (Profile)**：
  - 顶部展示大头像和复制 ID 功能。
  - 列表展示各项设置，包含：聊天室、管理功能（可直接作为我们 `admin-client` 的秘密入口）、修改手机、密钥设置、房卡包等。

## Impact
- Affected specs: 登录页面、大厅路由、用户基础状态管理。
- Affected code:
  - `src/App.tsx` (重构未进入房间 `!isJoined` 时的 UI 结构)
  - `public/images/ui/` (需要补充或复用现有的头像、图标素材)

## ADDED Requirements
### Requirement: 经典棋牌大厅导航
系统 SHALL 提供底部三栏导航的沉浸式大厅，让玩家感觉是在玩原生 App。

#### Scenario: 玩家打开网页
- **WHEN** 玩家访问网页根目录
- **THEN** 显示“大厅”网格，包含个人房卡信息和 6 个游戏大类入口；底部有“大厅”、“开房”、“我的”三个切换按钮。

### Requirement: 游戏入口与建房流程衔接
- **WHEN** 玩家在大厅点击“斗牛集合”
- **THEN** 弹出一个高质感的对话框，允许玩家输入自己的大名和 6 位房号进行加入或创建。

## MODIFIED Requirements
### Requirement: 管理后台入口隐蔽化
在“我的” Tab 的列表中增加“管理功能”开关，点击后如果密码正确，可跳转至控制台。
