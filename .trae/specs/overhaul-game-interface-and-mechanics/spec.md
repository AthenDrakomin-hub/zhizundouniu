# 游戏对局界面与机制彻底重构 Spec

## Why
当前进入游戏后的对局界面极其简陋（“白茫茫”的一片，像是默认的测试组件），毫无棋牌游戏的沉浸感和商业化质感。更严重的是，作为一个“积分制”棋牌游戏，玩家的初始积分错误地设置成了 1000 分（应为 0 分），并且缺失了基础的用户头像、声音控制开关等核心要素，完全无法匹配前序高质量的登录大门设计。

## What Changes
- **BREAKING**: 服务端玩家初始加入房间的积分从 1000 修改为 0。
- **游戏主桌面大换血**: 彻底废弃白底容器，将对局桌面替换为高清深色棋牌桌布材质，并调整玩家座位的排布，匹配真实赌桌的暗金色/红色氛围。
- **玩家卡片重构**: 为每个座位添加真实的玩家头像（Avatar），废除原有的简陋方块。
- **增加音效开关面板**: 在界面右上角/左上角增加音效控制组件，允许玩家一键开启/关闭游戏内的音效。
- **游戏流程 UI 优化**: 将“等待开局”、“准备开始”等按钮进行高质感（拟物/毛玻璃）重构，匹配当前的商业风格。

## Impact
- Affected specs: 房间加入逻辑、前端游戏主场景渲染、音效管理器。
- Affected code:
  - `src/server/game.ts` (玩家初始化逻辑)
  - `src/App.tsx` (游戏桌面、玩家座位布局、音效开关组件)

## ADDED Requirements
### Requirement: 积分与音效控制
系统 SHALL 在玩家进入时给予 0 分初始积分，并提供可持久化记忆的全局音效开关组件。

#### Scenario: 玩家进入游戏
- **WHEN** 玩家通过大门输入大名和房号进入对局桌面
- **THEN** 界面呈现高质量的暗色棋牌桌布背景，玩家的积分为 0，显示默认高档头像，且右上角有音效控制开关。
