# 游戏前端重构与管理功能分离 Spec

## Why
当前游戏客户端（前端）UI 过于像一个数据仪表盘（Dashboard），而不是一个沉浸式的棋牌游戏界面。此外，客户端中混合了大量的“管理后台”功能（如房间设置、添加机器人、房卡状态面板等），这违背了“前后端分离”和“玩家端/管理端分离”的商业安全逻辑，导致界面冗杂且容易被玩家窥探到管理功能。

## What Changes
- **移除前端管理功能**: 
  - 彻底删除前端（`src/App.tsx`）中的“管理后台”按钮及对应弹窗。
  - 彻底删除前端的“房间设置”按钮（齿轮图标）及 `RoomSettingsModal` 组件。
  - 彻底删除“添加机器人”按钮。
  - **BREAKING**: 房间的规则和设置未来将完全由 `admin-client` 在创建时或后台控制，前端玩家无法再修改房间规则。
- **UI 沉浸感升级**:
  - 移除前端界面中类似管理面板的“房卡状态”和“当前规则”卡片式大区块（右下角的大方块）。
  - 简化左上角的“当前局数”和“我的积分”显示，使其更像游戏内的信息条而非数据看板。
  - 清理硬编码的背景颜色（如 `bg-[#1c1c1e]`、`bg-slate-900`），确保之前全局配置的“发财红”主题（`radial-gradient`）和桌面背景图能够完美穿透并铺满全屏。

## Impact
- Affected specs: 客户端 UI 交互，玩家端权限（剥离房主特权配置功能）。
- Affected code:
  - `src/App.tsx` (主要重构)
  - `src/index.css` (如有残留底色一并清理)

## REMOVED Requirements
### Requirement: 玩家端（房主）配置房间规则
**Reason**: 违背前后端分离与商业防范逻辑，玩家端不应具备后台管理能力。
**Migration**: 玩家加入房间后只能进行“准备/抢庄/下注/摊牌”等纯游戏操作。

### Requirement: 玩家端添加机器人
**Reason**: 同上，机器人应由真实服务器或管理员后台统筹。
**Migration**: 移除该功能入口。

## MODIFIED Requirements
### Requirement: 游戏大厅与等待界面
将原本像后台数据表一样的 UI 结构，扁平化、游戏化。仅保留最核心的游戏操作按钮（准备开始、强制开局等）。
